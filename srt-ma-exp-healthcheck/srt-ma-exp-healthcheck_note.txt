#รายงานการบำรุงรักษา (ข้อ 3.7):
##ชื่อโครงการ ชื่อผู้รับจ้าง งวดงาน คาบเวลาการบำรุงรักษา วันที่และเวลาที่ปฏิบัติงาน
##รายชื่อผู้ปฏิบัติงานทั้งหมด
##บัญชีสรุปรายการอุปกรณ์และซอฟต์แวร์ที่บำรุงรักษาและผลการบำรุงรักษาแต่ละรายการ
##รายละเอียดผลการปฏิบัติงาน ปัญหาหรือสิ่งผิดปกติที่ตรวจพบ ข้อเสนอแนะ และแนวทางแก้ไข
##ลายมือชื่อผู้ปฏิบัติงานและผู้ควบคุมโครงการ


#รายงานการเฝ้าระวังภัยคุกคามทางสารสนเทศ (ข้อ 3.13):
##ชื่อโครงการ ชื่อผู้รับจ้าง งวดงาน คาบเวลา และรายชื่อผู้ปฏิบัติงาน
##รายละเอียดวิธีการตรวจสอบและการดำเนินงานแต่ละรายการ พร้อมลายมือชื่อ
##บัญชีสรุปรายการอุปกรณ์/ซอฟต์แวร์ที่เฝ้าระวัง ผลการเฝ้าระวัง ข้อสังเกต หรือแนวทางป้องกัน



3. หัวข้อทางเทคนิคสำหรับ Oracle RAC 19c (Preventive Checklist)
ตามขอบเขตงานที่ต้องดูแลความพร้อมใช้ตลอด 24 ชั่วโมง
 ควรมีหัวข้อดังนี้:
Clusterware & Instance Health: ตรวจสอบสถานะของ Grid Infrastructure และสถานะของ Node ใน Cluster ทุกโหนด
Storage & ASM: ตรวจสอบพื้นที่ว่างและการทำงานของ ASM Disk Groups
Database Health Check:
ตรวจสอบ Alert Log สำหรับข้อผิดพลาด (Critical Errors)
ตรวจสอบ Tablespace Usage (ป้องกันการเต็มของพื้นที่จัดเก็บข้อมูล)
ตรวจสอบ Backup Status (ความสำเร็จของการสำรองข้อมูล)
Performance Monitoring: ตรวจสอบการใช้งาน CPU, Memory และ Wait Events ที่ผิดปกติ


4. หัวข้อทางเทคนิคสำหรับ MSSQL (Preventive Checklist)
Service & Instance Status: ตรวจสอบสถานะของ SQL Server Service และ SQL Server Agent
Database Integrity: ผลการรัน DBCC CHECKDB เพื่อตรวจสอบความสมบูรณ์ของฐานข้อมูล
Maintenance Jobs: ตรวจสอบสถานะการทำงานของ Job ต่าง ๆ เช่น การทำ Index Rebuild/Organize และการ Update Statistics
Log Review: ตรวจสอบ SQL Server Error Log และ Windows Event Log ที่เกี่ยวข้องกับฐานข้อมูล


5. การเฝ้าระวังภัยคุกคามและ Patch (อ้างอิงข้อ 3.11 และ 3.13)
Security Updates: รายงานการตรวจสอบและติดตั้ง Security Patch หรือ Service Pack (ถ้ามีการดำเนินการในงวดนั้น)
Vulnerability Assessment: สรุปผลการเฝ้าระวังภัยคุกคามที่อาจส่งผลต่อฐานข้อมูล
